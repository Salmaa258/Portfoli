Git

És una eina de control de versions, que permet controlar, revisar, tornar enrere, etc, en les versions dels programes i projectes que anem desenvolupant. El seu objectiu principal es gestionar tots aquests canvis durant un període de temps.

Vinculació Git - Netbeans

Creació d’un projecte a Gitlab

Primer de tot, per poder connectar un projecte de netbeans a Git haurem de registrarnos a Gitlab amb el mateix usuari i contrasenya que tenim al centre. Un cop registrats, ens sortirà la següent imatge.

Farem clic a “crea un projecte” i seguidament a “crea un projecte en blanc”.

Declarem un nom al projecte, en aquest cas li posem “prova”. Hem de tenir en compte la URL del projecte ja que després la necessitarem per fer la vinculació.

Un cop acabat, ens sortirà una pàgina com la següent on podrem veure el nostre projecte.

Vincluació d’un projecte de Netbeans a Gitlab

Per vincular un projecte de Netbeans a Gitlab obrim el nostre Netbeans i ens dirigim al nostre menú “Team - Git - Clone”.

Se'ns obrirà la següent pestanya on haurem de posar la URL del nostre projecte de Gitlab, a mès del nostre usuari i contrasenya.

Li donem a “next” fins finalitzar. Ens sortirà un missatge dient si volem crear un projecte de Netbeans nou, li diem que si i el creem. L’anomenem “Git_prova”.  

Fem clic dret al projecte, “Git - Add”.

Seguidament, fem el mateix però li donem a commit. Ens sortirà una pestanya on podem veure els diferents registres del projecte.

Per últim, fem clic a “Git - remote - push” per poder connectar-ho al Gitlab.

Un cop fet, refresquem la pagina del Gitlab i veiem com ens surt el nom del projecte connecta't del Netbeans.

Comprovem que ens surt el mateix .java que tenim al Netbeans.

Creació de Commits

Per crear commits desde Gitlab, obrirem el nostre fitxer i li donarem a editar. Fem els canvis convenients (afegir un comentari “comentari 2”) i li posem nom al commit “Commit 2”. Finalment desem els canvis.

Farem un altre commit (“Commit 3”) per poder comparar-los més endavant.

Visualitzem els comentaris afegits.

Commit 2. Visualització del comentari afegit.

Commit 3. Visualització del comentari afegit.

Per veure tots els commits creats, anirem a l’apartat esquerre del nostre Gitlab on posa “Repository - Commits”.

Visualització dels canvis a Netbeans

Per aplicar aquests canvis al fitxer de Netbeans, farem clic dret sobre el fitxer, “Git - Remote - Pull”.

Un cop fet, veure’m els comentaris afegits en Gitlab en el document de Netbeans.

Creació de Branques

Per crear branques fem el mateix que anteriorment. Serveixen per crear diferents versions i després poder fusionar-les amb un merge. Establim un nom “Branch1”.

 

Desde el nostre Netbeans, si editem el fitxer i creem la branca, tindrem dues branques sobre aquest fitxer, la main i la nova creada “Branch1”. Un cop tenim les dues, podem comparar-les desde Gitlab amb un merge per fusionar-les.

Comparació de Commits

Creació de Tags

Per poder comparar diferents versions gràcies als commits que hem anat fent, haurem de crear tags per a cada commit per facilitar la comparació. Per això, farem clic sobre els commits, i li donarem a “Options - Tag”.

Ens sortirà una pestanya on haurem de declarar el nom del tag. Posem “Tag 2”.

Fem el mateix amb l’altre commit i l’anomenem “Tag 1”.

Un cop creats els tags, anirem a “Repository - Compare” i seleccionarem els dos tags que volem comparar.

Un cop comparats ens sortiran les diferències entre el primer commit (Tag2) i el segon (Tag1).

Creació de merges

Per això anem a “Repository - merge” i selecionem les dues branques que volem comparar/fusionar.

(Per la creació dels merges haurem de tenir dues branques mínim).

Establim un nom al merge i acceptem.

D’aquesta manera, com he mencionat anteriorment, podrem fusionar les dues versions.
